**File 1: Create** `pwa/src/app/order/[id]/pay/page.tsx`

This is a CLIENT component that initiates payment. Minimal UI — its main job is to call create-escrow and redirect.

```tsx
'use client';

import { useParams, useRouter } from 'next/navigation';
import { useState, useEffect } from 'react';
```

**Page flow:**
1. Read orderId from params: `const { id } = useParams<{ id: string }>();`
2. Read query params from URL: `subtotal`, `phone`, `comboDiscount` (passed when navigating to this page)
3. On mount, call POST /api/payments/create-escrow with the order data
4. On success, redirect to `confirmationUrl` (YooKassa payment page): `window.location.href = data.confirmationUrl`
5. Show a loading spinner while the API call is in progress
6. On error, show error message with a "Try again" button

**UI (minimal, functional per D-05):**
- Centered container, max-w-md
- Loading state: spinner icon + "Перенаправление на страницу оплаты..."
- Error state: red alert box + error text + "Попробовать снова" button
- Use Tailwind classes. Brand primary color: `bg-[#2d35a8]` for buttons

Note: If subtotal/phone are not in query params, show a form to enter them. But the primary flow is to receive them as query params from the order creation flow.

Create `pwa/src/app/order/[id]/confirm/page.tsx`.

This page handles confirmation for BOTH customer and supplier. Role is determined by the `role` query parameter and `token` query parameter.

```tsx
'use client';

import { useParams, useSearchParams } from 'next/navigation';
import { useState, useEffect } from 'react';
```

**URL format:** `/order/{id}/confirm?role=customer&token={jwt}`

**Page flow:**
1. Read params: orderId from URL path, role and token from search params
2. Validate: if role or token missing, show error "Недействительная ссылка подтверждения"
3. Fetch order data on mount: `GET /api/orders/${id}`
4. Show order summary: work_type, address, amount (total for customer, payout_amount for supplier)
5. Show confirmation button: "Подтвердить выполнение работ"
6. On click:
   ```typescript
   const res = await fetch(`/api/orders/${id}/confirm`, {
     method: 'POST',
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify({ token, role }),
   });
   const data = await res.json();
   ```
7. Handle response states:
   - `status === 'confirmed'` and `bothConfirmed === true`: Show full success — "Обе стороны подтвердили! Оплата будет переведена исполнителю."
   - `status === 'confirmed'` and `bothConfirmed === false`: Show partial success — "Вы подтвердили выполнение работ. Ожидаем подтверждения от {other party}."
   - `status === 'already_confirmed'`: Show — "Вы уже подтвердили этот заказ."
   - 401 error: Show — "Ссылка недействительна или истекла"
   - 409 error: Show — "Заказ не в статусе подтверждения"

**Pre-confirmation state:**
- White card on light gray background, max-w-md, centered, rounded-xl, shadow-lg

- Role badge: "Вы: Заказчик" or "Вы: Исполнитель" in pill badge
- Order summary section:
  - Тип работ: {work_type}

- Divider line
- Confirmation button: full-width, bg-[#2d35a8] text-white, rounded-lg, py-3, text-lg font-semibold
  - Text: "Подтвердить выполнение работ"
  - hover:bg-[#1a1f5c] transition-colors duration-200
  - cursor-pointer

**Post-confirmation state:**
- Same card layout
- Replace button with status message
- Both confirmed: green checkmark circle + success text + "Оплата будет переведена исполнителю в течение 1-3 рабочих дней"
- One confirmed: yellow clock icon + waiting text

**Error state:**
- Same card layout
- Red warning icon + error text + "Обратитесь в поддержку" contact link

**Loading state:**
