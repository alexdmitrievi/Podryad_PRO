Как исполнители попадают в систему?

Вы упомянули 3 канала: Telegram, MAX, PWA-форма. Как они должны работать вместе?

A

Единая форма, 3 точки входа

Одна и та же анкета исполнителя доступна через:
1) PWA — страница /join с полной формой
2) Telegram-бот — команда /start запускает пошаговый опрос (имя → специальность → город → телефон → контакт)
3) MAX-бот — аналогичный пошаговый опрос

Все данные попадают в единую таблицу contractors в Supabase. Админ видит всех в одном списке с пометкой канала.

B

PWA-форма основная, мессенджеры для записи контакта

Полная форма только в PWA (/join). В Telegram/MAX бот принимает минимум данных (имя, телефон, специальность) и отправляет ссылку на полную форму в PWA для дозаполнения. Мессенджеры — канал привлечения, PWA — канал регистрации.

C

PWA + n8n-автоматизация (рекомендую)

PWA-форма (/join) — основная точка входа с полной анкетой. Telegram/MAX боты собирают базовые данные через пошаговый диалог, n8n автоматически создаёт запись в Supabase и шлёт уведомление админу. Бот НЕ требует дозаполнения в PWA — данных из мессенджера достаточно для первого контакта. Админ дозаполняет профиль после личного разговора.
